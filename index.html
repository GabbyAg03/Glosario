GLOSARIO

-Branch:

es una línea de desarrollo separada dentro de tu repositorio. Puedes tener diferentes ramas para probar nuevas ideas sin afectar el código principal.

-Commit:

es como una foto instantánea de tu proyecto en un momento específico.

-CSS:

(Cascading Style Sheets) se utiliza para la presentación visual, es decir, el diseño y estilo de la página.

-GIT:

es un sistema que guarda el historial de cambios de tus archivos.

-HTML:

(Hyper Text Markup Language) es el lenguaje de marcado estándar utilizado para crear páginas web. A través de etiquetas y atributos, HTML define la estructura y el contenido de una página

-IDE:

es una herramienta que reúne todo lo que necesitas para escribir, probar y depurar tu código en un solo lugar.

-JavaScript:

añade interactividad y comportamiento dinámico a las páginas web.

-Prompt:

es una instrucción o entrada que se proporciona a un modelo de lenguaje para generar una respuesta o completar una tarea.

-Repositorio:

es como una carpeta donde Git guarda todo el historial de cambios de tu proyecto. Puede estar en tu computadora (local) o en un servidor en línea (remoto) como GitHub.

-Sprint:

es un período de tiempo corto y definido, generalmente de una a cuatro semanas, durante el cual un equipo se enfoca en completar una cantidad específica de trabajo.
